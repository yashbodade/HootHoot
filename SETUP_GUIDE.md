## Quick Start (5 minutes)

### For Local Development

```bash
# 1. Clone
git clone https://github.com/yashbodade/HootHoot.git
cd HootHoot

# 2. Install
pnpm install

# 3. Set environment (use Neon as fallback for local development)
cp .env.example .env.development.local

# 4. Run dev server
pnpm dev

# 5. Open browser
# → http://localhost:3000
```

### For Production (Vercel + AWS)

```bash
# Push to main branch
git push origin main

# Vercel automatically:
# 1. Installs dependencies
# 2. Builds Next.js app
# 3. Deploys to edge network
# 4. Runs instrumentation.ts → creates Aurora tables
# 5. App goes live at hoot-hoot.vercel.app
```

**No additional setup needed.** AWS env vars are injected automatically by the Vercel integration.

## Database Setup

### Auto-Migration on Vercel Deploy

On first cold start after deploy:
1. `src/instrumentation.ts` runs
2. Executes full schema from `src/app/api/aurora/migrate/_schema.ts`
3. Creates all 16 tables + 42 indexes
4. Tables are `CREATE TABLE IF NOT EXISTS` — safe to run repeatedly

**No manual migration commands needed.**

### Manual Migration (Local)

If you have direct Aurora access:

```bash
# Push Drizzle migrations
pnpm db:push

# Or run setup script
node scripts/setup-aurora.js
```

### Verify Database Connection

```bash
# Check Aurora status
curl https://hoot-hoot.vercel.app/api/aws/status

# Returns JSON with:
# - connected: true/false
# - backend: "AWS Aurora PostgreSQL"
# - pgVersion: "17.7"
# - tableCount: 16
# - indexCount: 42
# - tables: [...] with row counts
```

## Troubleshooting

### Build Errors

#### Error: `Cannot find module 'next'`

```bash
# Fix: Reinstall dependencies
rm -rf node_modules pnpm-lock.yaml
pnpm install
```

#### Error: `Error: ENOENT: no such file or directory`

```bash
# Fix: Check file paths (use absolute paths)
# Ensure all files exist in src/
ls src/
```

### Runtime Errors

#### Error: `502 Bad Gateway`

- **Cause:** Database connection failed
- **Fix:** Check `AWS_APG_PGHOST` in Vercel env vars
- **Verify:** `curl https://hoot-hoot.vercel.app/api/aws/status` should show connected

#### Error: `Auth failed - session not found`

- **Cause:** Cookie was cleared or session expired
- **Fix:** Sign in again
- **Debug:** Check browser DevTools → Application → Cookies for `hh_session`

#### Error: `Email not sent`

- **Cause:** SMTP credentials invalid
- **Fix:** Update `SMTP_*` env vars in Vercel
- **Test:** `curl -X POST https://hoot-hoot.vercel.app/api/test-email -d '{"email":"your@email.com"}'`

#### Error: `Game score rejected`

- **Cause:** Anti-cheat validation failed (score impossible)
- **Fix:** Try easier difficulty or verify score calculation
- **Debug:** Check console logs for validation errors

#### Error: `Fullscreen warning during arena`

- **Cause:** Browser exited fullscreen (F11 key, Alt+Tab, etc.)
- **Fix:** Press F11 again or press `F` in game to re-enter fullscreen
- **Note:** 3+ warnings auto-disqualifies

### Database Issues

#### Error: `No database connection available`

- **Cause:** `AWS_APG_PGHOST` not set
- **Fix (Local):** Set `DATABASE_URL` in `.env.development.local`
- **Fix (Production):** Vercel integration not connected — reconnect in Project Settings

#### Error: `Tables not created`

- **Cause:** Migration didn't run
- **Fix:** 
  ```bash
  # Local
  pnpm db:push
  
  # Or trigger via API
  curl -X POST https://hoot-hoot.vercel.app/api/aurora/migrate \
    -H "Authorization: Bearer admin-key"
  ```

#### Error: `Query timeout`

- **Cause:** Database overloaded or network latency
- **Fix:** Check Aurora CPU/memory in AWS console
- **Optimize:** Add indexes or cache more aggressively

### Deployment Issues

#### Error: `Build failed on Vercel`

1. Check Vercel build logs: https://vercel.com/dashboard
2. Common causes:
   - Missing env vars — add in Project Settings → Environment Variables
   - TypeScript errors — run `npx tsc --noEmit` locally
   - Missing dependencies — check `package.json`

#### Error: `Deployment stuck on "Building..."`

- **Fix:** Cancel and redeploy
  ```bash
  vercel --prod --force
  ```

#### Error: `CORS errors in browser console`

- **Cause:** Image domains not whitelisted
- **Fix:** Add to `next.config.ts` → `images.remotePatterns`
- **Example:**
  ```typescript
  {
    protocol: "https",
    hostname: "example.com"
  }
  ```

## Performance Optimization

### Caching Strategy

1. **Static content:** 30 days (fonts, logos)
2. **Game images:** 1 day (regenerate if updated)
3. **Leaderboard:** 60 seconds (DynamoDB cache)
4. **User sessions:** 30 days (DynamoDB cache)
5. **API responses:** SWR with revalidation

### Monitor Performance

```bash
# Check Core Web Vitals
curl https://hoot-hoot.vercel.app/api/aws/status | jq '.latencyMs'

# Expected: < 15ms local, < 100ms from Vercel
```
